# STS2 AI 控制台前端说明

这个页面是本地控制台，目标是让玩家能看懂当前状态、开关含义和数据是否正在正常写入。它仍是半成品界面，后续会继续补更细的解释、告警和演示模式。

## 页面区域

- 顶部状态：显示游戏连接、AI 接管、采集总开关和当前 Run。先看这里能快速判断系统是否正常。
- 实时采集动态：只显示最近几条动作，让玩家确认控制台正在读到游戏事件。完整记录仍在数据文件里。
- 左侧工作区：用于打开、收起或定位右侧模块。可以点击模块，也可以拖到右侧工作区。
- 战斗 AI：控制基础 AI 进程和自动出牌开关。
- LLM 模型接入：配置大模型接口，控制是否让 LLM 参与决策。
- 采集与训练：控制是否写入数据，并触发重建数据和重训。
- 右侧模块：展示 AI/LLM 决策、Run 数据、最近采集记录、策略评测和训练输出。

## 关键开关

| 开关 | 含义 | 建议 |
| --- | --- | --- |
| 允许 AI 出牌 | 让基础 AI 在战斗中自动执行出牌 | 演示自动战斗时打开 |
| 允许 AI 宏观操作 | 让 AI 处理地图、奖励、选卡、事件、休息点 | 初次演示建议关闭 |
| 采集总开关 | 是否写入战斗和宏观日志 | 正式采数据时打开，临时测试时可关闭 |
| 启用模型决策 | 是否请求 LLM | 不想消耗接口时关闭 |
| 动作选择 | LLM 如何给动作 | 推荐“只从合法候选动作里选” |
| 允许 LLM 自动战斗 | 是否执行 LLM 的战斗建议 | 先关着观察，再打开演示 |

## 新手流程

1. 启动游戏、Mod API 和控制台。
2. 打开控制台页面，点右上角“新手引导”。
3. 确认“游戏连接”不是未连接。
4. 如果只是看状态，先关闭“允许 AI 出牌”和“允许 LLM 自动战斗”。
5. 如果要采集数据，打开“采集总开关”。
6. 如果要演示基础 AI，打开“允许 AI 出牌”。
7. 如果要演示 LLM，先配置 Base URL、API Key 和 Model，测试连接后打开“启用模型决策”。
8. LLM 动作选择优先使用“推荐：只从合法候选动作里选”。
9. 观察右侧“AI 出牌逻辑”“LLM 决策”和“最近 Run”。
10. 训练前检查最近 Run 的质量标记，避免明显坏数据进入训练。

## 当前限制

- 控制台只负责展示和触发本地接口，不会替代游戏内判断。
- LLM 当前应作为决策脑使用，实际执行仍要走候选动作和合法性校验。
- 采集数据量还不足，训练效果会随样本质量明显波动。
- 宏观操作仍建议保守使用，尤其是商店和事件。
