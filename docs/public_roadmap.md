# 公开发展路线

本项目目标是做一个可采集、可训练、可复盘的 Slay the Spire 2 半自动 AI 工作区。当前版本已经具备基础工程能力，但仍处于早期阶段，最大短板是数据量和策略稳定性。

## 当前状态

| 模块 | 状态 | 说明 |
| --- | --- | --- |
| 游戏 Mod 接入 | 已跑通 | 可以通过本地 API 读取状态并执行动作 |
| 网页控制台 | 已跑通 | 可启动服务、查看 run、训练模型、配置 LLM |
| 数据采集 | 已跑通 | 支持人类/AI/LLM 日志采集 |
| 战斗 BC | 初版可用 | 样本量不足，只能作为 baseline |
| 宏观 BC | 初版可用 | 样本更少，需要更多路线和奖励数据 |
| LLM 接入 | 初版可用 | 适合建议和受约束执行，不应自由构造动作 |
| 自动评测 | 待完善 | 需要稳定记录通过率、死亡楼层和错误类型 |

## 近期路线

| 阶段 | 目标 | 输出 |
| --- | --- | --- |
| P0 | 扩充真实数据 | 更多人类 run、失败 run、AI/LLM 日志 |
| P1 | 稳定数据质量 | run 体检、数据标签、坏数据过滤 |
| P2 | 改进战斗决策 | 候选动作评分、药水使用、危险回合兜底 |
| P3 | 改进宏观决策 | 路线、奖励、事件、营火和商店策略 |
| P4 | 建立评测闭环 | Act1 Boss 通过率、失败原因、回归测试 |
| P5 | 提升模型能力 | 更好的 BC、偏好数据、可选 RL 实验 |

## 当前最需要的贡献

| 类型 | 为什么重要 |
| --- | --- |
| 数据包 | 训练质量主要受样本量限制 |
| 失败日志 | 能定位模型死因和错误动作 |
| Bug 报告 | 自动操作最怕状态错读和动作错映射 |
| 使用反馈 | 控制台和启动器仍需要真实环境验证 |

## 能力边界

当前项目不是成品 AI，也不能保证稳定通关。更准确的定位是：

```text
一个已经接通游戏、采集和训练链路的早期 AI 工作区。
```

短期目标不是直接做出完美玩家，而是先把数据闭环、动作安全、评测指标和复盘工具做扎实。
