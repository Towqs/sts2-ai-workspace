# 公开发展路线

本项目目标是做一个可采集、可训练、可复盘的 Slay the Spire 2 半自动 AI 工作区。当前版本已经具备基础工程能力，重点已经从“能否跑通”转向“稳定产出高价值数据、评测模型并逐步微调策略”。

## 当前状态

| 模块 | 状态 | 说明 |
| --- | --- | --- |
| 游戏 Mod 接入 | 已跑通 | 可以通过本地 API 读取状态并执行动作 |
| 网页控制台 | 已跑通 | 可启动服务、查看 run、训练模型、配置 LLM |
| 数据采集 | 已跑通 | 支持人类/AI/LLM 日志采集 |
| 战斗 BC / 候选动作 | 初版可用 | 候选动作评分已经接入线上策略，仍是 baseline |
| 宏观 BC | 初版可用 | 样本更少，需要更多路线和奖励数据 |
| Ironclad 选牌 Shadow Mode | 已接入 | 对 card reward 的每张牌和 skip 打分，默认 shadow 记录，不接管旧策略 |
| 自训练 | 初版可用 | 支持固定 seed、探索参数、run 评分和入训门槛 |
| Phase 2A 微调 | 已接入 | 用 admitted self-play run 做 reward-weighted 候选动作微调；无合格 run 时安全跳过 |
| LLM 接入 | 初版可用 | 适合建议和受约束执行，不应自由构造动作 |
| 模型包切换 | 已跑通 | `ModelZoo` 可放公开演示模型，控制台可一键切换 |
| 自动评测 | 初版可用 | 可记录楼层、胜负、非法动作、数据健康和 self-play 入训原因 |

## 近期路线

| 阶段 | 目标 | 输出 |
| --- | --- | --- |
| P0 | 高价值数据 | 稀有卡关键局、失败复盘、固定 seed 诊断、admitted self-play run |
| P1 | 数据质量 | run 体检、数据标签、坏数据过滤、self-play 入训/拒绝原因 |
| P2 | 战斗策略 | 候选动作评分、药水使用、危险回合兜底、Phase 2A 微调 |
| P3 | 宏观策略 | 路线、奖励、事件、营火和商店策略 |
| P3a | 战士选牌 | deck summary、三套构筑模板、skip 评分和 shadow/active 切换 |
| P4 | 评测闭环 | Act1 Boss 通过率、死亡楼层、失败原因、固定 seed 回归 |
| P5 | 模型能力 | 更好的 BC、偏好数据、seed 池、后续 PPO/Actor-Critic 实验 |

## 当前公开演示包

仓库包含 `AI_Training/ModelZoo/demo_local_20260430/`，用于让新用户不训练也能试用一版早期模型。

| 项目 | 数量 |
| --- | ---: |
| 战斗 BC 样本 | 642 |
| 候选动作行 | 2948 |
| 宏观 BC 样本 | 528 |

它的定位是 demo baseline，不是稳定通关模型。后续每次公开模型包都应附带样本规模、数据来源、训练指标、是否包含 Phase 2A 微调产物和已知问题。

## 当前最需要的贡献

| 类型 | 为什么重要 |
| --- | --- |
| 高价值数据包 | 稀有卡、Boss 前后、失败复盘和固定 seed run 比普通重复局更重要 |
| 失败日志 | 能定位模型死因和错误动作 |
| Bug 报告 | 自动操作最怕状态错读和动作错映射 |
| 使用反馈 | 控制台和启动器仍需要真实环境验证 |
| 模型评测结果 | 需要记录不同模型包的通过率和死因，避免只凭感觉迭代 |

## 能力边界

当前项目不是成品 AI，也不能保证稳定通关。更准确的定位是：

```text
一个已经接通游戏、采集和训练链路的早期 AI 工作区。
```

短期目标不是直接做出完美玩家，而是先把高价值数据闭环、动作安全、评测指标、固定 seed 复盘和 Phase 2A 微调工具做扎实。
