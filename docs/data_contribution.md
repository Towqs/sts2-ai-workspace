# 数据贡献说明

本项目当前仍是半成品：代码框架、控制台、数据采集、BC 训练、自训练评测、Phase 2A 微调入口和 LLM 接入已经跑通，但模型能力还远远不够稳定。主要瓶颈不是模型名字，而是高价值实战数据和 admitted self-play run 太少。

如果你愿意帮忙，请优先贡献真实游玩数据和可复盘的失败局。数据不是越普通越好，稀有卡关键回合、Boss 前后、药水救命、固定 seed 死因和高进度 run 更有价值。

## 需要什么数据

优先级从高到低：

| 数据类型 | 价值 |
| --- | --- |
| 人类完整 run | 最有价值，能学习真实路线、选牌、战斗取舍 |
| 人类战斗动作 | 用于训练战斗出牌和药水时机 |
| 人类宏观动作 | 用于训练路线、奖励、事件、营火选择 |
| AI/LLM 运行日志 | 用于定位错误和构造纠错样本 |
| 失败 run | 也有价值，可以分析死亡楼层和错误类型 |
| 固定 seed 复盘 | 用于复现同一死因、比较模型版本和做回归 |
| admitted self-play run | 用于 Phase 2A reward-weighted 微调 |

数据不要求每局都通关。失败局很重要，但要通过 run 体检和标签筛选；A1 死亡 self-play run 默认用于复盘，不直接作为 Phase 2A 正样本。

## 如何打包

推荐使用网页控制台里的：

```text
一键打包数据库
```

打包后文件会出现在：

```text
Data_Packages/
```

文件名类似：

```text
sts2_dataset_package_YYYYMMDD_HHMMSS.zip
```

这个 zip 通常包含：

- 原始 jsonl 采集记录
- run 标签
- 丢弃列表
- 数据包摘要

不包含：

- API Key
- 本地模型配置
- 训练后的模型权重
- 虚拟环境

模型权重如果需要公开分享，不要塞进数据包。请整理成 `AI_Training/ModelZoo/<model_id>/` 模型包，并附上 `manifest.json`、样本规模和已知问题。普通数据贡献优先提交 `Data_Packages/` 里的 zip。

## 提交给维护者

请把 `Data_Packages/` 里的 zip 数据包提交给项目维护者。

推荐方式：

1. 在 GitHub Issue 里说明你采集的数据情况，并附上数据包下载链接。
2. 如果数据包很小，也可以通过 Pull Request 提交到维护者指定的数据分支或附件位置。
3. 如果数据包较大，请使用网盘或其他文件分享方式，把链接发给维护者。

如果不方便使用 GitHub，也可以通过 QQ 联系维护者：

```text
2775089081
```

提交时请附上这些信息：

```text
游戏版本：
Mod 版本或提交号：
采集日期：
大概 run 数：
主要角色：
是否包含 AI/LLM 自动操作：
是否手动标记过丢弃 run：
备注：
```

## 不要提交什么

请不要提交：

- `AI_Training/model_config.json`
- API Key
- `.venv/`
- 本地快捷方式
- 游戏本体文件
- 明显包含私人路径、账号信息或无关日志的文件

如果不确定能不能提交，优先只提交控制台生成的数据包。

## 当前数据目标

短期目标：

| 目标 | 说明 |
| --- | --- |
| 100+ 人类战斗 run | 先让战斗 BC 有基本可学样本 |
| 50+ 人类宏观 run | 改善路线、奖励、事件和营火选择 |
| 20+ 固定 seed 复盘 run | 用于分析自动策略的稳定死因 |
| 10+ admitted self-play run | 用于触发 Phase 2A 微调 |
| 多个角色样本 | 避免模型只适配单一卡组节奏 |
| 模型包评测反馈 | 记录 demo/自训练模型在哪些 seed、楼层和场景失败，帮助决定下一轮训练方向 |

达到这些目标前，项目的自动 AI 都应被视为实验功能，不应宣称稳定通关。
